Чек – лист практического задания по разделу №3 «Тестовая документация»

Цель задания: составить чек-лист, тест-кейс, выполнить тестирование по подготовленным документам,  предоставить отчет о тестировании.

Шаг 1. На основе материалов лекции и предоставленных шаблонов, подготовить чек- лист проверки предложенного объекта тестирования или части его функционала.

Чек-лист — это список проверок необходимых для проведения тестирования ПО.

Структура чек-листа:

проверка
результат
комментарии
Основные правила при составлении чек-листа:
1.    Каждый пункт включает только одну операции
2.    Пункты для однозначной трактовки лучше начинать с существительного (Проверка, добавление и т.д.)
3.    Последовательность пунктов составляется исходя из логики ПО
Пример чек-листа: https://docs.google.com/spreadsheets/d/1MAfgNZ9qAYbNlH-NkfJ8pHtcUDwACXdsjOcX8yxYk1U/edit?usp=sharing
 

Шаг 2. На основе материалов лекции и предоставленных шаблонов, подготовить тест-кейсы проверки предложенного объекта тестирования или части его функционала.

 

Тест кейс — это документ, описывающий совокупность шагов, конкретных условий или параметров, необходимых для проверки тестируемой функции или ее части.

Основные свойства тест кейса:

Тест кейс должен быть независим от других тест кейсов;
В тест кейсе должна быть четкая формулировка шагов и ожидаемого результата;
В тест кейсе содержится вся информация для его выполнения;
Тест кейс должен быть выполнимый
Структура тест-кейса:

Предварительные условия (pre-condition) — шаги, которые необходимо выполнить перед началом тестирования по этому тест-кейсу.
Шаги (steps) — точная последовательность действий для выполнения проверки.
Шаги должны быть четкими и понятными. Их нужно писать так, чтобы понял даже человек, который видит проект и тестирование в первый раз.

Ожидаемый результат (expected result) — что мы получаем после выполнения шагов.
Приложения (attachments) — дополнительная информация, которая поможет выполнить тест-кейс, например, скриншоты, текстовые файлы и прочие файлы.
